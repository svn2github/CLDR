BCP 47 Extension U
draft-davis-u-langtag-ext-00

Abstract

This document specifies an Extension to BCP 47 which provides subtags that specify language and/or locale-based behavior or refinements to language tags, according to work done by the Unicode Consortium.

1.  Introduction

[BCP47] (Davis, M., Ed., “Tags for the Identification of Language (BCP47),” September 2009.) provides extensions identified by a single-character subtag for "the generation of identifiers that contain a language component and are compatible with applications that understand language tags". This document defines such an extension, using the letter 'u', to specify Unicode language/locale identifiers.

2.  BCP47 Required Information

Language tags, as defined by [BCP47] (Davis, M., Ed., “Tags for the Identification of Language (BCP47),” September 2009.), are useful for identifying the language of content. They are also used as locale identifiers (or can be mapped to locales) in many operating environments and APIs. However, most such locale identifiers also provide additional "tailorings" or options for specific values within a language, culture, or other regional variation. This extension provides a mechanism for using these additional tailorings within language tags for general interchange.

The maintaining authority for this extension's registry is the Unicode Consortium. Unicode defines common locale data and identifiers for this data:

ItemValue
Name Unicode Consortium
Contact Email cldr@unicode.org
Discussion List Email cldr-users@unicode.org
URL Location cldr.unicode.org
Specification Unicode Technical Standard #35 Unicode Locale Data Markup Language (LDML), http://unicode.org/reports/tr35/
Section Section 3.2 BCP 47 Tag Conversion

The specification of extension subtags is provided by Unicode Technical Standard #35 Unicode Locale Data Markup Language, Section 3.2 BCP 47 Tag Conversion (Davis, M., “Unicode Technical Standard #35: Locale Data Markup Language (LDML),” December 2007.) [LDML]. [LDML] (Davis, M., “Unicode Technical Standard #35: Locale Data Markup Language (LDML),” December 2007.) defines the formation of subtags which follow the language tag ABNF and other rules for the formation of language tags and subtags as defined in [BCP47] (Davis, M., Ed., “Tags for the Identification of Language (BCP47),” September 2009.). In particular, it specifies that subtags are restricted to the ASCII letters and digits, that case is not significant, and that subtags MUST NOT exceed eight characters in length.

[LDML] (Davis, M., “Unicode Technical Standard #35: Locale Data Markup Language (LDML),” December 2007.) specifies a canonical representation. LDML is available over the Internet and at no cost, and is available via a royalty-free license at http://unicode.org/copyright.html. LDML is versioned, and each version of LDML is numbered, dated, and stable. Extension subtags, once defined by LDML, are never retracted or change in meaning in a substantial way. LDML will include in a separate section the registration form in Section 2.2 (Registration Form).

2.1.  Summary

[LDML] (Davis, M., “Unicode Technical Standard #35: Locale Data Markup Language (LDML),” December 2007.) defines the subtags within the 'u' extension as a series of key-value pairs. Each key has a length of two characters, and each value has a length of three or more characters. Each key is followed by zero or one value. Case is not significant. The canonical case is lowercase; the canonical order of the keys is alphabetical.

The exact keys and values available, and their meanings, are defined in TODO ldml-registry TODO (, “Registry for Common Locale Data Repository tag elements,” September 2009.) [ldml‑registry].
