CLDR Beta 1.0 Comparison Charts

2003.10.31

The Common Locale Data Repository comparison charts show the comparison between data from different sources. The files are organized by locale. In each file, the first three columns identify the data item, while the subsequent columns contain data. The Common data is in the first data column, with other data sources following (where available). The latter sources are generated with public APIs.

The columns are color-coded, such as:

N. ParentNode Name ID COMMON (xml) LINUX (xml) OPEN_OFFICE (xml) WINDOWS (xml)

The successive rows provide a comparison of the data, with the following color coding:

  • Missing data is indicated by a white cell
  • Data that is the same as some column to the left uses the same color (with an equals sign in the cell)
  • Data that is identical except for a case change is indicated with a dagger
  • Data that is different than any column to the left has the column's normal color.

For example:

16 monthNames month 6 Junie = = =

78 territories territory ZA     South Africa Suid Afrika

Field Formats

The format of many of the fields in the comparison chart will be clear from the Name and ID, such as the months of the year. The format for others, such as the date or time formats, is structured and requires more interpretation. If you have any questions about the format, consult the overview in Data Formats. See especially the notes on Collation.

This is a beta version of the data, so there are known errors and omissions. If something looks odd to you, it may be a known error; you may consult this list to see if it is covered.

Filing Bug Reports

We request that people review the data for languages and countries that they are familiar with, and file bug reports where they see problems.  Please also review the known errors and omissions before filing a bug report.

Each bug report should list the following information:

  • The specific locale.
  • The date the comparison chart was generated (at the bottom of the chart).
  • The line(s) of data that are incorrect, and the data source (Common, Linux,...).
  • If one of the other data sources is correct, then that data source, otherwise the correct text to use.
  • The justification for the change.

Please group all bugs for a single locale into a single bug report.

Bugs may also be filed on the supplemental data, which is only available in XML format. Note that the default currency for a given country is in the supplemental data, not in the locale data. Similarly, the timezones for different regions are to be derived from Olson data, and are not in the locale data. The localized names for currencies and timezones, on the other hand, will be in the locale data (if available).

We have tried to make the format of the comparison charts as useful as possible. If you have suggestions for improvements, please also file it as a bug report.
